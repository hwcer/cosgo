## 注意事项

### 1. 懒淘汰特性
- 跳表大小可能暂时超过 `maxSize`
- 这是正常现象，当 dict 超出阈值时会触发清理
- **保证**：前 N 名始终是最高/最低分的元素

### 2. 分数为 0 的有效性
- 分数 0 是有效分数
- 可以被添加、查询、排名
- 不会因为分数为 0 而被特殊处理

### 3. 同分处理
- 相同分数的元素按插入时间排序
- 先插入的排在前面
- 不会覆盖或合并

### 4. 删除操作
- 删除守门员会自动触发守门员更新
- 删除非跳表元素只清理 dict
- 不会破坏跳表结构

## 总结

ZSet 模块通过以下设计实现高性能排行榜：

1. ✅ **跳表排序**：O(log N) 时间复杂度的增删查改
2. ✅ **同分保序**：分数相同时按插入时间排序
3. ✅ **守门员机制**：控制准入，保持榜单质量
4. ✅ **职责分离**：守门员管准入，裁剪模块管清理
5. ✅ **懒淘汰**：减少频繁删除，提升并发性能
6. ✅ **统一接口**：简化的 API 设计，降低维护成本

**适用场景**：
- 实时排行榜（游戏、社交）
- 优先级队列
- 延迟队列
- Top N 统计

**不适用场景**：
- 需要精确控制元素数量的场景
